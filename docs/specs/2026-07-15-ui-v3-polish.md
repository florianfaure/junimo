# Spec v3 — Polish UI-V2 : light-first, nav en tabs, junimo vivant

Date : 2026-07-15 · Auteur : Florian + Fable · Tâche : #38

## Contexte

Retour de Florian après le merge ui-v2 (#25–#35) : 12 points de polish et
d'évolution sur l'overlay. Exécution orchestrée : Fable pilote et dispatche
chaque tâche à un agent Haiku, Sonnet ou Opus selon la complexité ; Fable
n'intervient lui-même que sur blocage ou fix conséquent, à coût token maîtrisé.

## Décisions

### 1. Fenêtre ancrée sous la menu bar, alignée tray

- Aujourd'hui : `move_window_constrained(Position::TrayCenter)`
  (`src-tauri/src/tray.rs:136`) — la fenêtre peut chevaucher la menu bar.
- Cible : la fenêtre s'ouvre **entièrement sous la menu bar**, alignée
  horizontalement sur l'icône du tray (comportement menubar-app classique).
  `Position::TrayBottomCenter` du positioner, ou calcul manuel si besoin —
  en conservant la garde anti-panic et le sauvetage hors-écran de #37.
- Done : au toggle, fenêtre sous la menu bar et alignée sur l'icône, y compris
  sur moniteur secondaire.

### 2. Light mode par défaut, dark en option

- Le light devient le thème par défaut et prioritaire. Un réglage
  « Apparence » (light / dark) dans la page Réglages ; on ne suit plus
  automatiquement le système.
- Implémentation : attribut `data-theme` sur la racine + tokens Astryx,
  suppression de la dépendance à `light-dark()` — ce qui **résout la dette
  #36** (baseline macOS 13.3 / Safari 16.4).
- Persistance dans `settings` (comme les autres réglages).

### 3. Thème « tech » monochrome

- Composants les plus petits et denses possibles ; palette limitée à des
  variantes de blanc / gris / noir — **jamais** de blanc ou noir purs.
- Les jauges gardent des couleurs (ok / warn / alert) mais très désaturées et
  subtiles — à retravailler.
- Source de vérité : `src/theme/junimoTheme.ts` + `astryx theme build`
  (le CSS généré `junimo.theme.css` ne s'édite jamais à la main).

### 4. Navigation en tabs sous le junimo

- Remplace les accordéons persistants de #28. 4 tabs, chacun avec un icône :
  - **Usage** — les jauges ;
  - **Chats** — conversations en cours et terminées, avec les infos clés
    (projet, statut, durée, tokens, modèle) ;
  - **Projects** — détail par projet : repo git, chemin du dossier, dates,
    activité récente ;
  - **System** — MCPs utilisés + santé, skills les plus utilisés, autres méta.
- Le tab actif est persisté (même mécanique que l'état accordéons actuel).

### 5. Settings

- L'icône réglages du header devient un **engrenage**.
- La page Réglages passe en 2 tabs : **Compte** / **Réglages** (réutilise le
  composant tabs de la nav principale).

### 6. Junimo classique pixel-perfect + variantes dérivées

- Références fournies par Florian (2 images pixel-art) : la pose classique
  canonique (corps vert arrondi, tige sur la tête, joues roses, petits
  membres) doit être reproduite **exactement** dans les silhouettes de
  `src/junimo/model.ts` ; la 2e référence (bras levés) sert de base à la pose
  « célébration » (décision 9).
- Les images n'étant pas dans le repo, **Fable transcrit lui-même les grilles
  de pixels** (il a vu les images) et fournit la matrice dans le detail de la
  tâche — l'agent exécutant n'a pas besoin de voir les images.
- Les autres formes (round, star, nouvelles) deviennent des **variantes
  dérivées** de cette base classique (même tête/visage, silhouette modifiée).

### 7. Plus de formes et d'accessoires

- Au moins 3 nouvelles formes et 4 nouveaux accessoires dans `model.ts`,
  exposés dans l'éditeur. Accessoires en calques comme aujourd'hui (#32).

### 8. Éditeur : hover-edit + auto-save

- Survol du junimo du header : affordance d'édition — un **mini bouton
  crayon en overlay** apparaît au hover (alternatives écartées : tooltip seul
  = invisible ; bouton toujours visible = bruit).
- Suppression du bouton « Enregistrer » de l'éditeur : toute modification est
  **persistée immédiatement** (debounce court ~400 ms), en conservant la
  garde anti-écrasement existante (`settingsOpenRef`).

### 9. Animations d'état du junimo

- Machine à états pilotée par le snapshot, rendue dans `JunimoSprite`
  (canvas, frames programmatiques — pas de lib d'animation externe) :
  - `run` : une conversation est en cours → le junimo court ;
  - `eat` : consommation de tokens → il mange des tokens ;
  - `play` : il joue avec des tokens (variation d'idle actif) ;
  - `celebrate` : un chat vient de se terminer → pose bras levés (réf. image 2) ;
  - `bored` : rien en cours → il s'ennuie (bâille, s'assoit, regarde autour).

### 10. Tray : junimo vectoriel + animation fin de chat

- macOS/Tauri n'accepte pas de SVG direct dans le tray : l'icône est
  redessinée en **SVG source**, rendue en PNG multi-résolution (@1x/@2x/@3x)
  à la génération → plus de pixelisation.
- À la fin d'un chat : courte animation (swap de frames via `set_icon`,
  ~2 s) puis retour à l'icône de repos. Le mode template et les teintes
  d'alerte existantes (`tray.rs`) sont conservés.
